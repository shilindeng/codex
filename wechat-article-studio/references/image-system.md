# 图片系统

## 四维参数

- `theme`：内容主题领域
- `style`：视觉风格与表现方式
- `type`：图片用途
- `mood`：整体氛围与情绪强度

## 推荐组合

- 科技 + 未来科技 + 封面图 + 专业理性
- 商业 + 杂志封面 + 信息图 + 高级克制
- 职场 + 极简扁平 + 正文插图 + 高能激励
- 教育 + 手绘插画 + 信息图 + 温暖治愈
- 品牌 + 高级摄影 + 封面图 + 高级克制

## 默认图片规划

- 封面图：只用于公众号封面和 `thumb_media_id`，不进入正文
- 信息图：优先放在总结段后，用于收束全文观点
- 正文插图：按 H2/H3 章节权重和信息密度插入 2~4 张

## Prompt 组成

每张图的 Prompt 默认包含：

- 文章标题
- 核心观点
- 目标读者
- 图片用途
- 主题 / 风格 / 类型 / 氛围
- 构图要求
- 禁止事项

## 后端策略

- 自动默认只选官方图片后端：`gemini-api` 优先，其次 `openai-image`
- `gemini-web` 仅在显式指定 `--provider gemini-web` 时启用
- 选择 `gemini-web` 前必须先完成用户同意检查

## `gemini-web` 说明

- 适合已有 Cookie / 浏览器会话的场景
- 使用前必须明确说明它是非官方 best-effort 方案
- 不应默认持久化原始 Cookie 字符串，除非用户明确要求
