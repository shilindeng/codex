# 评分细则

默认满分 `100`。当前默认评分档案为 `tech-balanced`，通过阈值 `86`（并且必须通过质量门槛 `quality_gates`）。

## 默认评分档案：`tech-balanced`

- 标题与开头价值 `10`
- 核心观点与结构 `14`
- 技术准确与证据 `16`
- 术语处理与专名渲染 `12`
- 论证与例子 `12`
- 可执行性 `10`
- 可读性与节奏 `8`
- 传播句与记忆点 `8`
- 去 AI 味 `6`
- 读者共鸣 `4`

硬门槛：

- `total_score >= 86`
- `技术准确与证据 >= 12/16`
- `术语处理与专名渲染 >= 8/12`
- `核心观点与结构 >= 10/14`
- `ai_smell_hits <= 2`
- `title_leak_free = true`
- `action_closure_present = true`

## 维度与权重

兼容保留旧版 `viral` 档案：

- 标题与开头爆点 `12`
- 核心观点与副观点 `10`
- 说服策略与论证多样性 `12`
- 情绪触发与刺痛感 `12`
- 金句与传播句密度 `10`
- 情感曲线与节奏 `8`
- 情感层次与共鸣 `8`
- 视角转化与认知增量 `8`
- 语言风格自然度 `10`
- 可信度与检索支撑 `10`

## 质量门槛（硬门槛）

- `total_score >= 88`
- `emotion_value_sentences >= 6`
- `pain_point_sentences >= 4`
- `signature_lines >= 3`
- `argument_modes >= 3`
- `perspective_shifts >= 2`
- `ai_smell_hits <= 2`
- `可信度与检索支撑 >= 6/10`

说明：正式发布（`publish`）会拦截任何未通过 `quality_gates` 的稿件。

## 定位

- `review` 是结构化编辑拆解：产出爆款拆解、情绪价值句式、刺痛句式、AI 味问题与改稿优先级。
- `score` 是启发式评分 + `quality_gates` 判定：驱动自动回炉和发布门槛。
- 两者互补：每轮必须先 `review` 再 `score`。

## 低分处理顺序

1. 开头爆点：首屏刺痛 + 结果预期 + 反差判断
2. 情绪价值与刺痛句：让读者被理解且被推动
3. 论证多样性与视角切换：案例/对比/步骤/数据至少三种，视角至少两次
4. 金句密度：至少 3 句可截图判断句
5. 去 AI 味与节奏：删模板连接词，短段落，句长有波动
6. 可信度：补来源、数据或可回溯依据（不自动附“参考来源”板块到正文）
