# 评分细则

默认满分 `100`。当前默认评分档案为 `viral`，通过阈值 `86`（并且必须通过质量门槛 `quality_gates`）。

## 维度与权重

- 标题与开头爆点 `10`
- 核心观点与副观点 `10`
- 说服策略与论证多样性 `10`
- 情绪触发与刺痛感 `10`
- 金句与传播句密度 `10`
- 互动参与与社交货币 `10`
- 情感曲线与节奏 `8`
- 情感层次与共鸣 `8`
- 视角转化与认知增量 `8`
- 语言风格自然度 `8`
- 可信度与检索支撑 `8`

互动维度重点看：

- 点赞：是否有文末升华句、可截图金句、低门槛获得感
- 评论：是否有可站队观点、直接提问、经验补充口
- 转发：是否有社交谈资、身份标签、可复述判断
- 峰终：中段是否存在情绪/认知峰值，结尾是否有收束与互动引导

## 质量门槛（硬门槛）

- `total_score >= 86`
- `interaction_score >= 6/10`
- `可信度与检索支撑 >= 5/8`
- `ai_smell_hits <= 2`
- `template_penalty_hits <= 2`
- `similarity_passed = true`
- `citation_policy_passed = true`
- `editorial_review_passed = true`

说明：正式发布（`publish`）会拦截任何未通过 `quality_gates` 的稿件。

## 定位

- `review` 是结构化编辑拆解：产出爆款拆解、情绪价值句式、刺痛句式、AI 味问题与改稿优先级。
- `score` 是启发式评分 + `quality_gates` 判定：驱动自动回炉和发布门槛。
- 两者互补：每轮必须先 `review` 再 `score`。

## 低分处理顺序

1. 反模板与去同质化：先消掉近期高频套路句、重复开头和重复结尾
2. 阅读欲望：首屏场景、问题或反差是否足够把人带进去
3. 互动设计：明确点赞点、评论点、转发谈资，只保留篇型真正需要的主目标
4. 论证多样性：案例/对比/数据/场景/步骤是否足够支撑判断
5. 引用策略：正文只保留关键节点轻引用 `[1][2]`，完整来源放文末参考资料卡片
6. 结尾自然度：分析稿收束到判断，教程稿只留最少动作，不要万能清单
