# 图片 Prompt 体系

## 设计目标

- 用户显式参数优先
- 用户未指定时，先做文章级视觉分析，再做图片级 prompt 组合
- 默认正文图优先走概念型 `正文插图`，而不是先验地转成流程图/线框图
- Codex App / `gpt-image-2` 路径默认要求所有图片都有可读中文短字，且必须符合公众号图文运营场景

## 文章级模块

每次 `plan-images` 会先得到文章级策略：

- `visual_direction`：整篇文章的视觉方向
- `style_family`：例如编辑评论、知识解释、叙事手绘、商业决策、自然生活、抽象趋势
- `content_mode`：`conceptual / narrative / structural / data`
- `type_bias`：正文更偏哪类图型
- `decision_reasoning`：为什么这样判

## 图片类型模块

- `封面图`：重第一眼记忆点、裁切安全区、氛围、主视觉和清晰中文标题字
- `正文插图`：重概念隐喻、对象特写、场景叙事、人物压力、编辑画面和 1 个可读中文关键词
- `对比图`：重左右对照、前后对照、双边张力和极短中文标签
- `流程图`：只在真实步骤/顺序/路径明确时使用
- `信息图`：只在总结、框架、清单、指标压缩场景使用，优先做成可收藏/可转发图
- `分隔图`：重节奏变化、主题连续和一个短判断

## Codex App / gpt-image-2 规则

- 所有 Codex 图片必须有 `required_text` 或等价的可见中文短字。
- 封面文字要大，正文图文字要少但可读，不能只写成“可选小字”。
- 正文图默认不是架构图、流程图、仪表板、UI 截图或蓝图。
- 只有文中真实存在步骤、SOP、执行路径时，才允许 `流程图`。
- 只有文中真实存在清单、指标、总结卡时，才允许 `信息图`。

## 风格家族模块

- `编辑评论`：编辑拼贴、杂志颗粒、观点张力
- `知识解释`：清晰解释、教学感、知识卡片
- `叙事手绘`：人物感、场景感、手绘温度
- `商业决策`：商务图解、策略判断、对比张力
- `自然生活`：自然有机、柔和叙事、生活方式感
- `抽象趋势`：抽象几何、趋势动势、符号化表达

## 差异化约束

同一篇文章内的 prompt 会强制加入这些约束：

- 不要每张图都复用同一套视觉语法
- 变化构图、景别、主物件和留白模式
- 若一张图已经是结构图，邻近图片优先回到隐喻/场景/对象表达

## 推荐人工检查点

在执行 `generate-images` 前，优先检查：

- `image-strategy.json`
- `image-outline.md`
- `prompts/images/*.md`

如果发现某篇文章仍然过度偏流程图/信息图，优先回看：

- 文章级 `content_mode` 是否判错
- 某章节是否被误判成真实结构化内容
- 文内是否存在 `<!-- image:type=... -->` 之类强制 directive
- `image_controls.layout_family` 是否被显式指定为 `process` / `dashboard`
